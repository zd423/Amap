# 更新日志

本文档用于记录 AMap Companion 的近期人工整理更新日志。服务端同步 GitHub Release 后，还会生成面向客户端的更新日志：

```text
update_server/public/CHANGELOG.md
```

## 2026-05-26 亮屏自启与资源进一步精简

- “开机自动启动”改为“开机或亮屏自动启动服务”：开启后会在系统重启、应用更新、车机亮屏或解锁时自动启动伴侣服务，相当于打开软件后点击“启动伴侣服务”。
- 修正导航掉头红绿灯方向：掉头图标和右侧掉头图标不再被误判为右转；导航黄灯方向异常时会优先跟随当前导航转向，避免误显示掉头。
- 转向图标统一优先使用更美观的 `sou*_night_a530` 资源，并删除普通 `sou*_night`、不可达的 50+ 环岛出口资源和备用 `turn_*.png`。
- `sou` 转向资源由约 `269KB` 精简到约 `59KB`；删除 `turn_*.png` 后，当前签名 APK 体积约 `515KB`。
- 电子眼悬浮窗扩展支持更多高德 `CAMERA_TYPE`：按高德原始 Widget 图标显示监控、闯红灯、公交道、应急车道、非机动车道、压线、违停、鸣笛、逆行、HOV 等提示，不再只显示限速/摄像头/红绿灯三类。
- 优化导航红绿灯方向兼容：导航态 `dir=0` 不再直接视为掉头，黄灯 3 秒会延续上一阶段绿灯/红灯记录到的方向，避免黄灯尾秒误显示掉头箭头。

## 2026-05-24 手动融合 PR #9 与灵动岛优化

- 手动融合 [#9](https://github.com/zuo-qirun/amap-companion/pull/9) 的主要改动，并保留当前版本已有的副屏、日志、自定义字体、DIY 资源和高德广播自动显示逻辑。
- 新增“灵动岛（测试）”悬浮窗样式：巡航时采用更紧凑的胶囊布局，车道线间隔缩小，转向提示、车道和红绿灯会按优先级显示。
- 旧 UI / 新 UI / 灵动岛统一使用电子眼、限速、红绿灯图标；修正电子眼距离后偶发多出限速数字的问题，限速数字会优先填入限速圆圈。
- 蓝色转向框改为两行显示：第一行道路名称，第二行转向图标和剩余距离；导航退出时会同步清除蓝色框和残留图标。
- 修正刚进入导航或导航状态切换后，蓝色转向框可能只恢复箭头、道路名和剩余距离仍保持隐藏的问题。
- 优化悬浮窗整体宽度稳定性：内容短暂变少时不会立刻缩小边框，避免高德广播字段忽多忽少导致整个悬浮窗宽度来回抖动。
- 蓝色转向框和红绿灯方向箭头优先使用高德原始 `sou*_night` 转向资源，50+ 扩展动作码不再直接映射到环岛出口资源，避免“进入高速”等场景误显示环岛数字箭头。
- 根据高德 `GuideInfoProtocolData.switchIcon()` 逻辑补充 `65 -> 4`、`66 -> 5` 的扩展动作兼容映射，减少进入/靠左靠右类动作显示成默认直行的情况。
- 主屏导航红绿灯补充方向标识，和副屏/仪表显示保持一致；倒计时去除 `s` 后缀，灵动岛红绿灯尺寸略微收紧。
- 进一步缩小红绿灯胶囊、方向箭头/圆点和倒计时字号，减少红绿灯区域对悬浮窗宽度和高度的占用。
- 红绿灯胶囊背景改为纯色半透明背景，避免透明度调低后文字与地图白色背景混在一起。
- 修正 8 车道等宽车道场景下，副屏/仪表悬浮窗靠近屏幕边缘时被自动向左挤的问题。
- 车头方向不再显示角度，改为“北/东北/东/东南/南/西南/西/西北”等文字方向；ETA 区域优化为距离、时间、到达时间和目的地分行展示。

## 2026-05-24 进入软件自动启动服务

- 新增“进入软件后自动启动服务”选项；打开 AMap Companion 主界面后会按用户已选择的主屏、副屏或高德广播自动显示设置静默启动伴侣服务。
- 去除车道信息区域的黑色卡片背景，仅保留车道图形显示。
- 美化导航和巡航红绿灯样式；巡航方向灯改为更大的圆形箭头徽章加倒计时，导航单灯保留状态灯点加倒计时。
- 修正调整副屏/仪表百分比大小后，车道信息没有跟随缩放的问题。
- 新增多红绿灯巡航箭头 DIY：可在 `/sdcard/amap_companion/diy/` 放置 `cruise_arrow_left/right/straight/uturn` 的 PNG、WebP 或 JPG 图片，软件会等比适配到固定箭头位。
- 统一单红绿灯状态圆点与多红绿灯箭头圆圈的视觉尺寸。
- 红绿灯胶囊背景从颜色到透明渐变调整为纯色半透明背景，降低透明度时更适合贴在地图上显示。
- 日志公共保存目录调整为 `/sdcard/amap_companion/log`，与 DIY 资源目录统一放在 `/sdcard/amap_companion` 下。
- 修正副屏位置调整按钮偶发无效的问题；位置按钮现在同步保存坐标并通过专用广播即时移动副屏窗口，不再依赖重建副屏。
- 新增自定义字体功能：在 `/sdcard/amap_companion/` 或 `/sdcard/amap_companion/diy/` 放置 `font.ttf/font.otf/custom_font.ttf/custom_font.otf` 后，主界面、悬浮窗、副屏和红绿灯会自动使用该字体。

## 2026-05-23 资源轻量化

- 清理未被当前车道显示逻辑引用的旧版/备用车道图片资源，仅保留实际动态加载的 `lane_pdf_0..48` 和通知图标。
- `drawable` 资源从 `224,560B` 降至 `101,973B`，减少约 `54.6%`；整个 `app/src/main/res` 从 `516,103B` 降至 `393,516B`，减少约 `23.8%`。
- 当前本地签名 APK 体积为 `422,723B`，约 `412.8KB`，应用包更轻量。
- 更新安装流程新增备用通道：PackageInstaller 被车机拦截后，会继续尝试 InstallerX / InstallerX Revived，再使用带真实 `content://` APK URI 的系统默认安装器，减少小鹏车机 `uri:null` 安装拦截导致的“解析包时出现问题”。

## 2026-05-22 高德广播自动显示与日志调试优化

- 新增“高德广播自动显示悬浮窗”开关：开启后会通过前台服务在后台持续监听高德广播，收到高德地图广播后自动显示悬浮窗，不再依赖“使用情况访问权限”。
- 主界面“启动/关闭悬浮窗”改为“启动/关闭伴侣服务”，主屏悬浮窗和副屏悬浮窗改为独立显示位置选项；手动启动和高德广播自动显示都会跟随用户选择。
- 修正“关闭伴侣服务”在高德广播自动显示开启时无法真正停止的问题；关闭时会同时关闭主屏、副屏和广播自动显示监听。
- 日志页将 `READ_LOGS` 和存储权限拆分为两个授权入口；保存日志只根据 `/sdcard/amap_log` 是否可写决定是否使用公共目录，不再要求同时具备日志读取权限。
- 日志可在应用内查看、复制或保存；优先保存到 `/sdcard/amap_log`，无权限时回退到应用私有日志目录，方便反馈和排查问题。
- 去除车道区域的“车道信息”标题，只保留车道图形；悬浮窗和副屏最小缩放下限调整为 `30%`。

## 2026-05-23 副屏仪表稳定与通知弹窗修复

- Fix companion service stop behavior：伴侣服务关闭逻辑会同步停止主屏/副屏显示与高德广播自动显示监听，避免服务残留。
- 修正副屏悬浮窗在“导航/巡航退出隐藏仪表”开启时可能被高德打开阶段误隐藏的问题；高德广播自动显示触发后，副屏会跟随用户的副屏选项启动。
- 修正副屏悬浮窗反复开启/关闭时可能出现大小不一致的问题；副屏重建会强制重新读取当前“副屏大小”，启动伴侣服务也不再自动改动主屏显示选项。
- 修正切换主屏/副屏悬浮窗开关时可能触发前台服务启动超时并闪退的问题；关闭选项时不再先启动服务。
- 修正副屏面板尺寸计算混用主屏密度的问题；副屏 Presentation 构建和红绿灯胶囊现在固定使用副屏 Context 的密度，避免大小一会儿像主屏、一会儿像副屏。
- 修正副屏仪表面板在部分车机/投屏 display 上尺寸异常的问题：副屏面板会按仪表屏分辨率限制最大绘制比例，并避免新 UI 左右分栏在 1080p 仪表屏上重叠。
- 修正进入 AMap Companion 主界面时可能自动启动前台服务，导致系统弹出“是否允许发送通知”并临时拦截触摸的问题；现在只有用户点击“启动伴侣服务”、开机后台监听或收到对应广播时才启动服务。
- 安装/覆盖更新后不再通过 `MY_PACKAGE_REPLACED` 自动拉起前台服务，避免刚安装完成就触发通知权限弹窗；开机自动监听仍按用户开关保留。
- 软件内发现新版本弹窗改为可滚动 Markdown 渲染，支持标题、列表、代码块和行内反引号，更新日志阅读更清晰。
- 解除副屏仪表面板按 DPI/分辨率计算的最大缩放限制，“副屏大小”滑杆会按用户设置的百分比直接生效。
- “下载已改高德”入口更新到新的定制地图 Release，APK 文件名改为英文并保留镜像站下载选项。
